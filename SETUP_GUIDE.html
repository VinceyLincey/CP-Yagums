YAGUMS — Setup Guide

Complete folder structure, database setup, and profile picture configuration.

1 Project Folder Structure

Place all files inside XAMPP's htdocs folder, in a subfolder called yagums:

C:/xampp/htdocs/yagums/ ← your project root │ ├── config.php ← DB credentials + helpers ├── setup.php ← run ONCE, then delete │ ├── api/ │ ├── login.php │ ├── register.php │ ├── profile.php ← info / password / avatar │ ├── announcements.php ← post + vote │ └── admin_log.php │ ├── uploads/ ← created automatically by setup.php │ ├── .htaccess ← blocks PHP execution (security) │ └── avatars/ │ ├── 2.jpg ← user_id 2's profile picture │ ├── 6.png ← user_id 6's profile picture │ └── ...etc │ ├── index.html ← landing page ├── login.html ├── register.html ├── profile.html ├── dashboard-student.html ├── dashboard-admin.html ├── dashboard-lecturer.html ├── dashboard-maintenance.html ├── shared.css └── notifications.js
⚠️ Never put PHP files inside the uploads/ folder. The .htaccess created by setup.php blocks this.

2 Database Setup

Open phpMyAdmin at http://localhost/phpmyadmin and follow these steps:

  1. Click New in the left panel
  2. Database name: yagums → Collation: utf8mb4_0900_ai_ci → Click Create
  3. Click the yagums database in the left panel
  4. Click Import → Choose file → select yagums_final.sql → Click Go

3 Run Setup (First Time Only)

After importing the database, visit:

http://localhost/yagums/setup.php

This will:

  • Create the uploads/avatars/ folder automatically
  • Generate a 10-digit numeric backup code for the Super Admin
  • Display the code once — copy and store it safely
🗑️ Delete or rename setup.php immediately after copying the backup code. Anyone with the URL can regenerate it.

5 Profile Picture Upload

Profile pictures are uploaded via api/profile.php using multipart form data. The file is saved as:

uploads/avatars/{user_id}.jpg ← or .png .webp .gif

Rules enforced by the API:

  • Max size: 2 MB
  • Allowed types: JPEG, PNG, WebP, GIF
  • Old picture is automatically replaced (no duplicates)
  • Each user only has access to their own picture via their session token

Frontend upload example (from profile.html):

// JavaScript fetch with FormData const form = new FormData(); form.append('avatar', fileInput.files[0]); fetch('api/profile.php', { method: 'POST', headers: { 'Authorization': 'Bearer ' + token }, body: form // no Content-Type header — browser sets multipart automatically }) .then(r => r.json()) .then(data => { if (data.success) { img.src = data.profile_picture + '?v=' + Date.now(); } });

6 Booking Privacy

✅ All booking queries are scoped to the logged-in user's session token. A student can only ever see their own bookings — the API enforces WHERE user_id = {token.user_id} for all non-admin roles. Admins and Facility Managers can view all bookings.
